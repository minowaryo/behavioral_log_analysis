# /adr — ADR作成コマンド

新しいArchitecture Decision Record（ADR）を作成します。

## 手順

1. `docs/adr/` 内の既存ADRを確認して次の番号を特定する
2. `docs/adr/ADR-XXXX-[title].md` として新規ファイルを作成する
3. 以下のテンプレートで記入する

## テンプレート

```markdown
# ADR-XXXX: [意思決定のタイトル]

## Status
Proposed

## Date
[YYYY-MM-DD]

## Context
[なぜこの判断が必要になったか。背景・問題・制約を記載]

## Decision
[何を決めたか。具体的に記載]

## Rationale
[なぜそう決めたか。採用しなかった代替案も記載]

### 採用しなかった代替案
- **[代替案1]**: [採用しなかった理由]
- **[代替案2]**: [採用しなかった理由]

## Consequences
[この決定による影響・トレードオフ]

### メリット
-

### デメリット・リスク
-

## Related
- [関連ADR・ドキュメントへのリンク]
```

## ADRを書くべきタイミング

- 新しいライブラリ・フレームワークの採用
- アーキテクチャパターンの変更
- セキュリティ方針の変更
- DBスキーマの大規模変更
- AI開発ポリシーの変更
