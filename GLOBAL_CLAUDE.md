# Global Claude Code Memory

## 言語・スタイル設定
- 返答は日本語で行うこと
- コードコメントは英語、説明文は日本語
- コミットメッセージは英語

## Git 操作ルール
- SSH 接続を使用すること（`git@github.com:...`）

## ドキュメント管理ルール（必須）

Planモードで設計・決定した内容は、以下の3ファイルに必ず記録・更新すること。
実装開始前にこれらのファイルを読み込み、内容を把握した上で作業に入ること。

| ファイル | 用途 | 記載内容 |
|---------|------|---------|
| `PLAN.md` | 開発計画 | フェーズ計画・実装対象・完了条件・PoC成功条件 |
| `BACKGROUND.md` | システム背景 | 導入背景・課題・方針・位置づけ |

### ルール詳細

1. **Planモード終了時**: 決定した計画・仕様・背景を該当ファイルに必ず反映する
2. **タスク開始時**: 2ファイルをすべて読み込んでからコードに着手する
3. **仕様変更時**: 実装と同時に該当ファイルを更新する（ドキュメントと実装を乖離させない）
4. **ファイルが存在しない場合**: 最初のPlanモード完了時に新規作成する

## よく使うパターン
- 監査ログは JSONL 形式で `logs/audit.jsonl` に記録
