# ADR-0008: Claude API を AI 分析エンジンとして採用する

## Status
Accepted

## Date
2026-04-12

## Context

5つの観点（自動回顧録・学習フィルター・メンタルトリガー検知・AIコーチング・ビジネスアイデア提案）でのログ分析（F-004〜F-007, F-011）に使用する AI エンジンを選定する必要があった。

選択肢:
1. **Claude API (Anthropic)**: Sonnet / Opus モデル
2. **OpenAI ChatGPT API (GPT-4o)**: GPT シリーズ
3. **ローカル LLM (Ollama 等)**: ローカル実行

## Decision

**Claude API (Anthropic)** を採用する（モデル: claude-sonnet-4-6 をデフォルト）。

## Rationale

| 観点 | Claude API | GPT-4o | ローカル LLM |
|---|---|---|---|
| 長文コンテキスト | 200K tokens | 128K tokens | モデル依存（多くは 8K〜32K） |
| 日本語品質 | 高 | 高 | 中〜高（モデル依存） |
| プライバシー | Anthropic サーバー | OpenAI サーバー | 完全ローカル |
| コスト | 従量課金 | 従量課金 | ほぼ無料（ハードウェア費用） |
| セットアップ | API キーのみ | API キーのみ | GPU・モデルダウンロード必要 |
| 安定性 | 高 | 高 | ハードウェア依存 |

- 2ヶ月分のログ（60日×複数エントリ）を一括で分析するため、長いコンテキストウィンドウが必要
- ローカル LLM は初期セットアップコストが高く MVP に不向き
- Claude はユーザー自身も利用中のサービスであり、出力スタイルへの信頼がある

**プロンプト設計方針**:
- システムプロンプトに分析観点の定義と出力フォーマット（JSON）を明示する
- ユーザーの個人情報（氏名・メール）はプロンプトに含めない
- プロンプトキャッシュを活用してコストを削減する

## Consequences

- Anthropic API キーを `.env` で管理する（`CLAUDE_API_KEY`）
- モデルは `claude-sonnet-4-6` をデフォルトとし、`.env` で切り替え可能にする
- 分析コストはログ量に比例するため、将来的にコスト監視が必要
- API エラー時は最大3回リトライし、それでも失敗した場合はエラーログに記録して次回スケジュールで再実行する
