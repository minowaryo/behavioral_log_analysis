# ADR-0006: OpenAI Whisper API を音声文字起こしに採用する

## Status
Accepted

## Date
2026-04-12

## Context

音声日記機能（F-001）では、ブラウザで録音した音声ファイルをテキストに変換する必要がある。
変換方式として以下の選択肢があった:

1. **OpenAI Whisper API**: クラウドベースの音声認識 API
2. **ブラウザの Web Speech API**: クライアントサイドでリアルタイム認識
3. **音声ファイルのまま保存（テキスト変換なし）**: Claude API に直接音声を渡す

## Decision

**OpenAI Whisper API** を採用する。

## Rationale

| 観点 | Whisper API | Web Speech API | 音声ファイルのまま |
|---|---|---|---|
| 日本語精度 | 高（多言語対応） | 環境依存（Safari等で差異大） | N/A |
| オフライン | 不可 | 一部対応 | 不要 |
| iOS Safari 対応 | ○（サーバー側処理） | △（リアルタイム認識が不安定） | ○ |
| テキスト検索 | ○（DB保存） | ○（DB保存） | ✗ |
| Claude API 連携 | ○（テキストで渡せる） | ○ | △（音声対応が限定的） |
| コスト | $0.006/分 | 無料 | 無料 |

- Web Speech API は iOS Safari でのリアルタイム認識が不安定で、本プロジェクトの主要環境（スマートフォンブラウザ）に適さない
- 音声をテキスト化してDBに保存することで、Claude API による分析が容易になる
- Whisper はサーバー側で処理するため iOS 等のブラウザ差異の影響を受けない

## Consequences

- OpenAI API キーの管理が必要（`.env` で管理）
- 1回の録音につき API コストが発生する（音声 5分で約 $0.03）
- 音声ファイルは一時的にサーバーに保存してから Whisper に送信する設計が必要
- 最大 25MB / 音声ファイルの制限がある（約 30〜60分録音相当）
