# acceptance-criteria.md — 受け入れ基準

> 各機能のリリース判定基準を定義します。
> QAテストの基準としても使用します。

---

## 受け入れ基準テンプレート（Given-When-Then形式）

---

### AC-001: [機能名]

**関連要件**: F-001 / UC-001

```
Given: [前提状態]
When:  [操作]
Then:  [期待する結果]
```

例:
```
Given: 管理者がログイン済みである
When:  ユーザー一覧ページにアクセスする
Then:  全ユーザーのリストが表示される
  And: 各ユーザーの名前・メール・ステータスが表示される
  And: ページネーションが機能する
```

## 共通受け入れ基準（全機能共通）

### セキュリティ
- [ ] 未認証ユーザーは保護されたリソースにアクセスできない
- [ ] 権限のないユーザーは403を受け取る
- [ ] 入力バリデーションエラーは422を返す

### パフォーマンス
- [ ] 一覧APIは 200ms 以内に応答する（P95）
- [ ] N+1クエリが発生していない

### テスト
- [ ] Feature Testがすべての基準をカバーしている
- [ ] テストがCIで通過している
