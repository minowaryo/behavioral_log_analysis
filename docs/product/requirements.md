# requirements.md — 要件定義

> 対象: 人間レビュー用（高レベル）
> 更新者: ビジネス側・BA

---

## 1. 背景・目的

日々の行動・思考・学習内容を記録し続けることで、自己成長の可視化や精神状態の把握、新たなアイデア創出を支援するパーソナルツールを作りたい。

現在、音声での日記記録と YouTube の視聴という2つの習慣があるが、これらのデータは蓄積されるだけで振り返りや分析に活かせていない。Claude API を用いた AI が定期的にこれらのログを多角的に分析することで、自己理解と成長を促進する。

---

## 2. スコープ

### IN（対象）
- 音声日記の録音・文字起こし・保存（毎日）
- YouTube 視聴履歴の自動取得（YouTube Data API v3 / OAuth2）
- 日次ログの蓄積・閲覧（JSON 形式で保存）
- 6つの観点での AI 定期分析（週次・2ヶ月ごと）
- 分析結果の閲覧 UI
- モバイルファーストのレスポンシブ Web UI

### OUT（対象外・MVP）
- ネイティブモバイルアプリ（iOS / Android）— レスポンシブ Web で代替
- Twitter / Instagram 等他の SNS 連携
- リアルタイム通知・プッシュ通知
- PWA（将来検討）
- 複数ユーザーの管理画面（将来対応、設計は見越す）

---

## 3. Actor（利用者）

| Actor | 説明 | 権限レベル |
|---|---|---|
| ユーザー（自分） | ログを記録・閲覧・分析結果を受け取る | 全機能利用可 |
| システム（スケジューラ） | 定期的に分析ジョブを自動実行する | 分析ジョブ実行のみ |
| Claude API | ログを受け取って分析テキストを返す | 外部サービス |

## 4. 機能一覧

| 機能ID | 機能名 | 概要 | Actor | 優先度 |
|---|---|---|---|---|
| F-001 | 音声日記録音 | ブラウザから録音 → Whisper API で文字起こし → 保存 | ユーザー | 高 |
| F-002 | YouTube 履歴取得 | Google OAuth2 認可後、YouTube Data API v3 で視聴履歴を自動取得・保存 | ユーザー / システム | 高 |
| F-003 | 日次ログ管理 | 1日分のログ（音声日記＋YouTube履歴）をまとめて閲覧・JSON 保存 | ユーザー | 高 |
| F-004 | 自動回顧録分析 | 蓄積ログから自動で振り返りレポートを生成（Claude API） | システム | 高 |
| F-005 | 学習フィルター分析 | 学習傾向・興味分野を抽出・可視化（Claude API） | システム | 高 |
| F-006 | メンタルトリガー検知 | 気分・感情のパターンを検知してアラート（Claude API） | システム | 高 |
| F-007 | AI コーチング | 蓄積ログをもとにアドバイス・目標提示（Claude API） | システム | 中 |
| F-008 | タイムカプセル | 過去の自分へのメッセージを将来の自分に届ける | ユーザー / システム | 中 |
| F-009 | 定期分析スケジューラ | 週次・2ヶ月ごとに自動で分析実行（Laravel Task Scheduler） | システム | 高 |
| F-010 | 認証 | Laravel Sanctum ベースのセッション認証（将来マルチユーザー対応） | ユーザー | 高 |
| F-011 | ビジネスアイデア提案 | YouTube視聴ジャンル × 日記の関心事 × 学習フィルター結果を掛け合わせた新規アイデア提案（Claude API） | システム | 中 |

## 5. 非機能要件

| 項目 | 要件 |
|---|---|
| 動作環境（優先） | スマートフォンブラウザ（iOS Safari / Android Chrome） |
| レスポンシブ対応 | Tailwind CSS でモバイルファースト設計（sm / md / lg ブレークポイント） |
| 音声録音 | MediaRecorder API（ブラウザ標準）。iOS Safari 対応を確認する |
| ローカル動作 | MVP はローカル環境で完全に動作すること（MySQL + storage/app/） |
| AWS 移行準備 | ストレージ・DB は AWS（S3 / RDS）へ移行できる設計にする |
| セキュリティ | API キー（Claude / Whisper / YouTube OAuth）は `.env` のみで管理 |
| 可用性 | MVP 段階ではローカル動作のみ（99.9% SLA は将来対応） |
| データ保持 | 音声ファイルは S3 互換ストレージに保存。テキスト・履歴は MySQL に保存 |
| 将来のマルチユーザー | DB テーブルには `user_id` を含めて設計する |

---

## 6. 制約・前提

- Google OAuth2 のクライアント ID / シークレットは Google Cloud Console で取得済みであること
- Claude API のキーは Anthropic Console で取得済みであること
- OpenAI の Whisper API キーは取得済みであること
- Laravel 11 / PHP 8.2 以上の環境を前提とする
