# Diary 画面（録音・日記） — 画面仕様メモ

## 対応UC
- UC-001: 音声日記を録音・保存する

## 画面構成

```
┌─────────────────────────────┐
│  [← 戻る]  Diary  [カレンダー]│  ← ヘッダー（モバイル）
├─────────────────────────────┤
│  2026年4月18日（今日）         │  ← 日付セレクター（タップで変更可）
├─────────────────────────────┤
│                             │
│       ◉ 録音ボタン            │  ← 80px以上の円形ボタン
│    （タップで録音開始/停止）    │
│                             │
│  状態: idle / recording /   │
│        processing / done   │
├─────────────────────────────┤
│  文字起こし結果テキストエリア    │  ← Whisper API 結果を表示・編集可
│  （編集可能、最大10,000文字）   │
├─────────────────────────────┤
│         [保存する]            │  ← 文字起こし完了後にアクティブ化
├─────────────────────────────┤
│  ── 今日の記録 ──             │
│  [エントリ1] 08:32 文字起こし… │  ← 当日の VoiceDiary 一覧
│  [エントリ2] 12:10 文字起こし… │
└─────────────────────────────┘
```

## 状態遷移

| 状態 | ボタン表示 | 説明 |
|---|---|---|
| idle | ● グレー | 録音待機中 |
| recording | ■ レッド + 波形アニメ | 録音中（再タップで停止） |
| processing | ⟳ グレー（スピナー） | Whisper API 処理中（操作不可） |
| done | ✓ グリーン | 文字起こし完了（テキスト編集・保存可） |

## 注意点・設計意図

- iOS Safari は `audio/webm` 非対応のため `audio/mp4 (AAC)` を使用する（UC-001 業務ルール）
- processing 中は録音ボタン・保存ボタンを無効化し、二重送信を防ぐ
- 1日に複数エントリを保存できる。既存エントリは画面下部に一覧表示する
- 文字起こし結果のテキストエリアは自動拡縮（最低5行・最大20行程度）
